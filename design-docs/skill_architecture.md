# Autonomous UX Process Skill Architecture

To build an AI agent harness capable of running the MVP UX process autonomously, the workflow has been broken down into a **4-skill architecture**. This architecture follows Anthropic's skill guidelines, ensuring each skill handles a specific, cohesive part of the workflow without becoming overly complex.

## Total Skills Needed: 4

### 1. `problem-and-hypothesis-analyzer`
**Purpose:** Combines Problem Definition (Step 1) and Hypothesis Formulation (Step 2).
**Description:** This skill analyzes the initial idea, identifies the core problem and target audience, and synthesizes critical user pain points. Once the problem is defined, it autonomously generates clear, testable Value-Proposition and Solution hypotheses. Grouping these steps ensures the hypotheses are directly rooted in the defined problem space.

### 2. `risk-reward-evaluator`
**Purpose:** Executes Risk-Reward Assessment (Step 3).
**Description:** This skill acts as a critical gatekeeper. It evaluates the hypotheses generated by the previous skill against potential risks (cost of being wrong) and rewards (value gained if correct). It is responsible for making the autonomous decision to prioritize high-reward ideas and ruthlessly pruning features that do not directly test the core hypotheses.

### 3. `mvp-core-architect`
**Purpose:** Defines the MVP Core (Step 4).
**Description:** Taking the validated hypotheses from the evaluator, this skill strips away unnecessary UX steps (like deep information architecture or storyboarding). It formulates a single, focused Goal Statement, identifies the absolute minimum Core User Stories required, anticipates edge cases, and outlines the Minimal Flow and data structure required for the AI coding phase.

### 4. `ai-coder-prompt-engineer`
**Purpose:** Prepares for AI-Assisted Coding (Step 5).
**Description:** This skill bridges the gap between design and development. It synthesizes the Goal Statement, Core User Stories, and Minimal Flow into a comprehensive, agentic prompt. It sets strict guidelines focusing on usability for the core flow, ensuring the subsequent coding agent builds an MVP that tests the idea effectively rather than failing due to a poor interface.
